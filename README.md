# PTCG ABC Card Viewer

ポケモンカード「AI Battle Challenge」のカードデータを、**ブラウザ内だけ**で閲覧・絞り込みするビューアです。Vite + React + TypeScript。

CSV（と任意でカード画像PDF）をアップロードすると、ブラウザ内で派生列（進化段階・フラグ・DPE など）を生成し、一覧／詳細／フィルター／比較トレイで閲覧できます。日本語・英語の両CSVに対応し、UIごと言語を切り替えられます。**データは一切サーバーへ送信しません。**

## 特長

- **アップロード式・データレス設計** — 起動後にCSV/PDFを選ぶだけ。リポジトリにもサーバーにも実データを置きません。
- **ブラウザ内で派生列を生成** — 生CSVから `has_ability` / `stage` / `prize_value` / `dpe`（エネルギー1個あたりダメージ）などを共有変換モジュールで算出。
- **日本語 / 英語の切り替え** — JP・EN 両方のCSVを読み込むと、ヘッダのトグルでデータとUIをまるごと切り替え。
- **リロードしても再選択不要** — アップロードしたCSV/PDFを IndexedDB に保存し、次回起動時に自動復元（「別ファイル」で破棄）。
- **強力な絞り込み** — 種別 / 進化段階 / タイプ / カテゴリ / 拡張 / フラグ（ex・メガ・ACE SPEC・テラ）/ 特性。各ファセットに件数を表示し、複数指定はAND、同一グループ内はORで併用。
- **2つの一覧 + 詳細** — グリッド／テーブル表示、全画面のカード詳細（特性・ワザ・弱点／抵抗・DPE など）。
- **比較トレイ** — 気になるカードを並べて画像／詳細で見比べ（最大12枚）。
- **PDFカード画像** — `Card_ID List_*.pdf` から該当ページを pdf.js で描画し、**周囲の余白とリンク文字を自動トリミング**してカード本体だけ表示。表示中ページだけを取得するため巨大PDFでも軽量。
- **お気に入り・検索・並び替え** — ★お気に入り、名前／HP／にげる／DPE 等での並び替え、名前・ワザ・特性などの横断検索。

## データの取り扱い

- コンペのCSV・PDFは**再配布禁止**です。**リポジトリには実データをコミットしません**（`.gitignore` で `*.csv` / `*.pdf` / `data/` 等をブロックし、合成ダミーのみ例外許可）。
- 読み込んだカードデータは**ブラウザのメモリと IndexedDB（ブラウザローカル）だけ**で扱い、外部へ送信しません。
- `localStorage` に保存するのは表示設定のみ（テーマ・表示モード・お気に入りID・比較トレイID・サイドバー幅）。カード内容そのものは保存しません。
- 実データを読み込んだ状態のまま**公開ホスティングしないでください**。

## 使い方

1. 起動するとアップロード画面が表示されます。
2. カードデータの **`.csv`**（と任意でカード画像の **`.pdf`**）を選択、またはドラッグ&ドロップします。
   - ファイル名に `JP` を含むPDFは日本語、それ以外は英語として扱われます。
   - 日本語・英語の両CSVを読み込むと、ヘッダ右上の**日本語 / English トグル**で切り替えられます。
3. 一覧からカードを選ぶと**全画面の詳細ビュー**が開きます（`←`/`→` で前後、もう一度クリックで閉じる）。
4. 一度読み込めば次回以降は**自動復元**されます。別のデータに差し替えるにはヘッダの**「別ファイル」**を押します。

## 画面構成

ツインレール構成のシングルビューです。

- **ヘッダ** — 検索／デッキ作成モード、読み込み枚数、言語トグル＋「別ファイル」、ライト／ダーク切替。
- **ツールバー** — 件数表示、横断検索、並び替え、お気に入りのみ表示、グリッド／テーブル切替。
- **左サイド（フィルター）** — 各ファセットを件数つきで表示。クリックで絞り込み、折りたたみ可。
- **中央（一覧）** — グリッド or テーブル。各カードにPDF由来のサムネイルを表示。
- **右サイド（比較トレイ）** — ★・📌 で集めたカードを画像／詳細で並べ見比べ（幅はドラッグで調整・保存）。
- **カード詳細** — 画像・基本ステータス・特性／ワザ・弱点／抵抗・カテゴリなどを全画面表示。

> デッキ作成モードのUIは準備中です（60枚・同名4枚まで・たねポケモン必須などのルール検証はドメインモデルとして実装済み）。

## データ前処理パイプライン（任意・ローカル専用）

アプリ本体はアップロードしたCSVをブラウザ内で処理するため、この前処理は**必須ではありません**。機械学習などで**1カード1行の特徴量テーブル**が欲しいときに使うローカル専用ツールです。

```bash
npm run build:data
```

`scripts/build-card-data.mjs` が `data/JP_Card_Data.csv`（ロング形式・日本語ヘッダ・1ワザ1行）を読み込み、`data/cards_processed.csv` を生成します。変換ロジックはアプリと共有の `src/lib/cardTransform.ts` を使用します。

**処理の要点**

- **正規化**: 前後空白を除去し、`n/a` / `None` / 空 などの欠損表記を空値に統一。
- **ワザ単位の解析**: `[特性]` 始まりは特性として攻撃と区別。コストは無色（`●`）と有色エネを数えて `cost_total` 等を算出。ダメージは `Static` / `Multiplier`（`×`）/ `Conditional`（先頭 `-`）/ `None` を判定。
- **DPE**: 攻撃ワザ かつ `cost_total > 0` かつ `Static` のときだけ `damage_value / cost_total`。それ以外は `null`。
- **集約**: カードIDでまとめ、フラグ（`ex` / `メガシンカex` / `ACE SPEC`）・進化段階・集約値・`moves_json` を1行に格納。
- **判定の方針**: 種別は雑多なカテゴリ列ではなく「段階種類」列で、フラグはカード名ではなく「ルール」列で判定。

出力列の概要（主要なもの）:

| 列 | 内容 |
| --- | --- |
| `card_id` / `name` / `expansion` / `collection_no` | 基本情報 |
| `kind` / `stage_label` / `stage` | 種別・段階表記・段階数値（たね=0 / 1進化=1 / 2進化=2） |
| `hp` / `type` / `weakness` / `resistance` / `retreat` | ステータス |
| `is_ex` / `is_mega_ex` / `is_ace_spec` / `prize_value` | フラグ・サイド枚数（メガ=3 / ex=2 / その他=1） |
| `has_ability` / `ability_count` / `n_moves` | 特性・ワザ数 |
| `best_dpe` / `peak_damage` | 最大DPE・最大ダメージ |
| `moves_json` | 全ワザ・特性のJSON配列 |

検証ログとして、ユニークカード数・種別別件数・段階別平均HP などのサマリを出力します（数値は入力データ依存）。

## プロジェクト構成

```
src/
  App.tsx                エントリ。未読込なら FileLoader、読込後は PokemonCardViewer
  main.tsx               React マウント
  viewer/
    PokemonCardViewer.tsx メインビュー（一覧・フィルター・比較・詳細）
    refModel.ts           Card → 表示用モデルへの変換
    i18n.ts               UI文言・値の日英辞書
    styleString.ts        インラインスタイルのヘルパー
  components/
    FileLoader.tsx        CSV/PDF アップロードUI
    CardImage.tsx         PDFページの描画・トリミング
  csv/
    loadCsv.ts            CSVパース（papaparse）
    foldMoves.ts          1ワザ1行 → 1カード1行へ折りたたみ＋派生列付与
    columnMap.ts          日本語/英語ヘッダのマッピング
  lib/
    cardTransform.ts      共有変換ロジック（アプリ／前処理スクリプト共通）
    cardStore.ts          アップロードファイルの IndexedDB 永続化
    energy.ts             エネルギー記号の解析
    deriveKind.ts         種別判定
    pdfRender.ts / pdfPages.ts  PDF描画・トリミング／ページ計算
    filter / sort / favorites / pinned / deck  絞り込み・並び替え・保存・デッキモデル
  styles/global.css       全体スタイル
scripts/
  build-card-data.mjs     前処理パイプライン（ローカル専用）
  check-data-safety.mjs   実データの誤コミット検査
  make-dummy-csv.ts       ダミーCSV生成
dummy/                    実データを含まない合成CSV（コミット可）
```

## 開発用ダミーデータ

実データを含まない合成CSVを生成できます（開発・スクリーンショット用）。

```bash
npm run make:dummy
```

生成物は `dummy/` に置かれ、**git にコミットしてよい唯一のCSV**です。日本語版 `dummy/dummy_jp.csv` と英語版 `dummy/dummy_en.csv` を用意しています。

## スクリプト一覧

```bash
npm run dev         # 開発サーバー
npm run build       # 型チェック + 本番ビルド
npm run preview     # ビルド結果のプレビュー
npm run test        # ユニットテスト（vitest）
npm run make:dummy  # ダミーCSV生成
npm run build:data  # 前処理済みCSVの生成（ローカル専用・任意）
npm run check:data  # 実データの誤コミット検査
```

`npm run check:data` は、合成ダミー以外の CSV/TSV/PDF がリポジトリ内に見つかると失敗します。
