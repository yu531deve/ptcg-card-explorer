# PTCG ABC Card Viewer

ポケモンカード「AI Battle Challenge」のカードデータを、**ローカル環境だけ**で閲覧・前処理するためのビューアです。Vite + React + TypeScript で動作します。

CSVを読み込んでカード一覧・詳細を表示し、PDFのカード画像を「カード部分だけ」切り出して表示します。右サイドバーでカードを一時保存して見比べたり（比較トレイ）、検索／デッキ作成のモード切替に対応します。さらに、機械学習向けの**特徴量テーブル（前処理済みCSV）**を生成するパイプラインを備えています。

## データの取り扱い（重要）

- コンペのCSV・PDFは**再配布禁止**です。リポジトリは非公開のまま運用し、コラボレーターはコンペ参加者のみに限定してください。
- カードデータは**ブラウザのメモリ上だけ**で扱います。外部サーバーには送信しません。localStorage に保存するのは**カードIDと表示設定だけ**（お気に入りIDの一覧・比較トレイのIDの一覧・サイドバー幅）で、カードの内容そのものは保存しません。
- コンペデータを載せた状態でこのアプリを**公開ホスティングしないでください**。

### data/ フォルダにファイルを置く

ローカルの `data/` フォルダにコンペのファイルを置くと、アプリ起動時に自動で読み込みます。Vite の小さな配信ミドルウェアが `data/` を `/data/*` として**HTTPレンジ対応**で配信するため、巨大なPDFでも**表示するページだけ**を取得します（全ダウンロードしません）。

`data/` に置くファイル名（JP単体運用の例）:

| ファイル | 用途 |
| --- | --- |
| `JP_Card_Data.csv` | 元データ（生CSV） |
| `cards_processed.csv` | 前処理済みCSV（`npm run build:data` で生成。アプリはこちらを優先的に読込） |
| `Card_ID List_JP.pdf` | カード画像PDF（詳細画面の画像表示に使用） |

> **注意**
> - `data/` は `npm run check:data` の検査対象から除外されています。
> - `Card_ID List_*.pdf` は非常に大きい（数百MB規模）ため、リポジトリが非公開でも**git にはコミットしない**ことを推奨します（履歴が肥大化します）。共有ドライブ等で各自配布してください。

## 使い方

1. `data/` にコンペファイルを置く（上の表を参照）。
2. `npm run build:data` で前処理済みCSV `data/cards_processed.csv` を生成する。
3. `npm run dev` でアプリを起動する。
   - `data/` にファイルがあれば**自動で読み込み**ます。
   - 無い場合は、画面のファイルピッカー／ドラッグ&ドロップで手動読み込みもできます。
4. 一覧からカードを選ぶと、**全画面の詳細ビュー**が開きます。
   - 一番上にカード画像、続いて各種フィールドを表示。
   - 「← 戻る」で一覧へ。「← 前のカード / 次のカード →」で移動（キーボードの `←` `→` `Esc` も対応）。
   - 特性持ちカードは「特性」と「ワザ」を分けて表示します。
   - ★ でお気に入り、📌 で比較トレイに追加できます。

### モードとサイドバー

- 画面右端に**全高固定のサイドバー**があります。仕切りをドラッグで幅変更、ボタンで開閉でき、幅は保存されます。
- 上部の切替で**検索モード／デッキ作成モード**を選べ、サイドバーの中身が入れ替わります。
  - **検索モード → 比較トレイ**: 📌 で追加したカードを並べて見比べます。「画像だけ（2列の画像ギャラリー）／詳細も表示（1列で画像＋情報）」をトグルで切替。
  - **デッキ作成モード → デッキパネル**: デッキのドメインモデル（60枚・同名4枚まで・たねポケモン必須などのルール検証）は実装済みで、カード追加UIは今後対応予定です。
- 「フィルター」「統計」セクションは見出しクリックで折りたためます。

## 主な機能

- **自動読み込み**: `data/cards_processed.csv` を優先的に読み込み（無ければ生CSVにフォールバック）。
- **2つの一覧表示**: テーブル表示／カードグリッド表示（どちらも仮想スクロール対応で高速）。各カードに**PDFから切り出したサムネイル画像**を表示。
- **検索**: カード名・全ワザ効果・ルールを NFKC 正規化して横断検索。
- **絞り込み（フィルター）**: 種別 / 進化段階・種類 / タイプ / カテゴリ / 拡張 / フラグ（ex・メガ・ACE SPEC・テラ）/ **特性あり** で絞り込み。複数フィルターはAND、同一グループ内はORで併用可能。フィルター・統計は折りたたみ可能。
- **お気に入り（★）／比較トレイ（📌）**: 一覧・詳細から登録。お気に入りのみ表示の切替、比較トレイでの並べ見比べに対応（IDのみ localStorage 保存）。
- **検索／デッキ作成モード**: 右サイドバーの内容を切替。デッキ作成のルール検証ロジックは実装済み（UIは準備中）。
- **並び替え**: `cardId` / `name` / `hp` / `retreat` / `damage` / `collectionNo`。
- **詳細表示**: 先頭にカード画像、全フィールド・ワザコストをエネルギーアイコン表示・特性／ワザの分離表示・DPE（エネルギー1個あたりダメージ）など。
- **カード画像表示**: PDF（`Card_ID List_*.pdf`）から該当カードのページを pdf.js で描画し、**周囲の白い余白とリンク文字を自動でトリミング**してカード本体だけを表示。表示中のページだけをレンジ取得するため巨大PDFでも軽量。
- **URL同期**: 検索・フィルター・並び順・表示モードをURLに反映（共有可能）。**カードデータはURLに含めません**。

## データ前処理パイプライン

```bash
npm run build:data
```

`scripts/build-card-data.mjs`（Node ESM）が、`data/JP_Card_Data.csv`（ロング形式・日本語ヘッダ・1ワザ1行）を読み込み、**1カード1行のクリーンな特徴量テーブル** `data/cards_processed.csv` を生成します。

### 処理の流れ

1. **正規化**: 列名・値の前後空白を除去し、`n/a` / `None` / 空 などの欠損表記を**本物の空値**に統一。
2. **ワザ単位の解析**: ワザ名のある行ごとに以下を算出。
   - `is_ability`: ワザ名が `[特性]` で始まれば特性（攻撃ワザではない）。
   - コスト解析: `●`（無色）と漢字エネルギー（草・炎・水・雷・超・闘・悪・鋼・竜・無）を1文字ずつ数え、`cost_total` / `cost_colorless` / `cost_typed` を算出。`0コスト`・空・記号以外は `cost_total = 0`。
   - ダメージ解析: `damage_value`（数値）と `damage_kind` を判定。
     - `×` を含む → `Multiplier`（倍率）
     - 先頭が `-` → `Conditional`（条件付き・軽減など）
     - 数値のみ → `Static`
     - ダメージ無し → `None`
   - `dpe`（damage per energy）: **攻撃ワザ かつ `cost_total > 0` かつ `Static` のときだけ** `damage_value / cost_total` を算出。倍率・条件付き・特性は嘘の効率値を出さないよう `null`。
3. **カード単位に集約**: 「カードID」でまとめ、静的属性・フラグ・集約値・`moves_json` を1行に格納。

### 判定で守っているルール（落とし穴対策）

- ポケモン/トレーナー/エネルギーの判定は、雑多で空の多い**カテゴリ列ではなく「段階種類」列**で行う（`ポケモン/...`→Pokemon、グッズ/サポート/ポケモンのどうぐ/スタジアム→Trainer、基本/特殊エネルギー→Energy）。
- `ex` / メガ / ACE SPEC は**カード名のマッチではなく「ルール」列**で判定（`ex` / `メガシンカex` / `ACE SPEC`）。
- 特性はワザ名の `[特性]` 始まりで判定。

### 検証ログ

実行すると件数の検証サマリを出力します。期待値:

```
Unique cards: 1267
ex: 121, Mega: 30, ACE SPEC: 29
Avg HP — たね 97 / 1進化 137 / 2進化 208
```

## 前処理済みCSVの列仕様

`data/cards_processed.csv` の主な列:

| 列 | 内容 |
| --- | --- |
| `card_id` / `name` / `expansion` / `collection_no` | 基本情報 |
| `category` / `rule` / `prev_stage` | カテゴリ・ルール・進化前 |
| `kind` | Pokemon / Trainer / Energy / Unknown |
| `stage_label` / `stage` | 段階表記（例 `ポケモン/たね`）と数値（たね=0 / 1進化=1 / 2進化=2、非ポケモンは空） |
| `hp` / `type` / `weakness` / `resistance` / `retreat` | ステータス（retreat はポケモンのみ数値、空は0／非ポケモンは空） |
| `is_ex` / `is_mega_ex` / `is_ace_spec` | フラグ |
| `prize_value` | サイド枚数（メガ=3 / ex=2 / その他=1） |
| `has_ability` / `ability_count` | 特性の有無・個数 |
| `has_weakness` / `has_resistance` | 弱点・抵抗力の有無 |
| `best_dpe` | そのカードの Static 攻撃の最大 DPE |
| `peak_damage` | 最大 `damage_value` |
| `n_moves` | ワザ数（特性は除く） |
| `moves_json` | そのカードの全ワザ・特性を `[{name, is_ability, cost, cost_total, damage_raw, damage_value, damage_kind, dpe, effect}, ...]` のJSON配列で格納 |

## プロジェクト構成

```
scripts/
  build-card-data.mjs    前処理パイプライン（CSV → 特徴量テーブル）
  check-data-safety.mjs  コンペデータの誤コミット検査
  make-dummy-csv.ts      ダミーCSV生成
src/
  csv/
    loadDataDir.ts       data/ からの自動読み込み（processed優先）
    loadProcessed.ts     前処理済みCSVのパース（moves_json展開）
    loadCsv.ts           生CSVのパース（フォールバック・手動読込用）
    columnMap.ts         日本語/英語ヘッダのマッピング
    foldMoves.ts         生CSVのワザ折りたたみ
  components/
    CardTable / CardGrid     一覧（テーブル／グリッド、サムネ付き）
    CardDetail / CardImage   詳細パネル／PDFページ描画・トリミング
    CardActions              ピン・お気に入りボタン（共通）
    FilterPanel / SummaryBar フィルター／統計
    SidePanel                右サイドバー（モードで内容切替）
    ComparePanel / DeckPanel 比較トレイ／デッキパネル
    FileLoader / EnergyPip   手動読込／エネルギー表示
  lib/
    filter / sort / energy   絞り込み・並び替え・エネルギー解析
    favorites / pinned       お気に入り・比較トレイの保存（IDのみ）
    deck                     デッキのルール・検証（ドメインモデル）
    pdfRender / pdfPages     PDF描画・トリミング／ページ計算
    labels / urlState        日本語ラベル／URL状態同期
vite.config.ts           data/ をレンジ対応で配信するミドルウェア
```
